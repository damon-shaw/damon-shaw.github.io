◆ ProcessGame()

state_t ProcessGame ( gamestate_t *  G,
bool  user_first,
bool  b1,
int  ms50,
bool  reinit 
)

This is the FSM for handling operations from both the user and the computer when the application is in GAME mode.

Parameters
GA game-state object that will be referenced for the game board
user_firstA boolean that dictates if the user will move first following initialization. When this is true, the user will go first; when this is false, the computer will go first.
b1A boolean that describes the pressed state of the top button on the Boost-XL board. When this is true, the button is pressed; when this is false, the button is not pressed.
ms50A boolean that describes the expired state of the 50 millisecond software timer. When this is true, the timer has expired; when this is false, the timer is still counting.
reinitA boolean that dictates if the mode must be reinitialized. This should be set to true when the master FSM transitions from IDLE mode to GAME mode.
Returns
state_t A state_t value that tells the master FSM which mode to process next. When the game should be processed next, return game. When the game is complete and we should return to idle mode, return idle.
